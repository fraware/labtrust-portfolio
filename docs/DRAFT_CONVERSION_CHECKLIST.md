# Draft conversion checklist (Eval → Draft)

This document specifies the **claim table** format and how it ties to the portfolio-wide Draft conversion checklist in `STATE_OF_THE_ART_CRITERIA.md` (section 2.1). Before marking a paper submission-ready or Draft-complete, run the **Phase 3 / hostile reviewer checklist** in [STATE_OF_THE_ART_CRITERIA.md](STATE_OF_THE_ART_CRITERIA.md#3-submission-readiness-phase-3--hostile-reviewer-checklist) (claim-evidence matrix, repro under 20 min, variance rules, no kernel redefinition, overclaim check, repro block lists every script). For polish standards (captions, repro block format, figure style), see [PAPER_ARTIFACT_STYLE.md](PAPER_ARTIFACT_STYLE.md).

## When a paper can move to Draft

A paper satisfies the Draft bar only when all five items in section 2.1 of `STATE_OF_THE_ART_CRITERIA.md` are met. Summary:

| # | Requirement | Location / format |
|---|-------------|-------------------|
| 1 | Claim table, 3–4 claims max | `papers/Px/claims.yaml` |
| 2 | Evidence mapping per claim | In `claims.yaml`: release_id, artifact_paths, table_id, figure_id |
| 3 | Two tables + one figure autogenerated from release | Scripts under `scripts/`; outputs in draft |
| 4 | Limitations (no overclaim, determinism levels, auditability) | Section in DRAFT.md |
| 5 | Repro block at top of draft | Exact commands at top of DRAFT.md |

## Claim table: `papers/Px/claims.yaml`

Each paper that targets Draft must have a `claims.yaml` file with **at most 3–4 claims**. Structure:

```yaml
# papers/Px/claims.yaml
paper_id: "P0_MADS-CPS"   # or P1_Contracts, etc.
title: "Short paper title"
release_id: "p0_e3_release"   # primary release backing the draft tables

claims:
  - id: "C1"
    statement: "One-sentence claim (falsifiable)."
    evidence:
      release_id: "p0_e3_release"
      artifact_paths:
        - "datasets/releases/p0_e3_release/release_manifest.json"
        - "kernel/mads/NORMATIVE.v0.1.md"
      table_ids: ["Table 1"]
      figure_ids: []

  - id: "C2"
    statement: "Second claim."
    evidence:
      release_id: "p0_e3_release"
      artifact_paths:
        - "datasets/runs/<run_dir>/conformance.json"
      table_ids: ["Table 1"]
      figure_ids: ["Figure 1"]
```

### Fields

- **paper_id**: Paper identifier (e.g. P0_MADS-CPS). Must match folder name under `papers/`.
- **title**: Short title for reference.
- **release_id**: Default release used when a claim does not override it (optional if every claim has its own release_id).
- **claims**: List of 3–4 claims. Each claim has:
  - **id**: C1, C2, C3, C4 (or equivalent).
  - **statement**: One-sentence falsifiable claim.
  - **evidence**: Object with:
    - **release_id**: Dataset release or run identifier (e.g. `p0_e3_release`, or a run dir name under `datasets/runs/`).
    - **artifact_paths**: List of paths (relative to repo root) to artifacts that back the claim (evidence bundle, conformance.json, eval summary, etc.).
    - **table_ids**: List of table labels used in the draft (e.g. `["Table 1", "Table 2"]`).
    - **figure_ids**: List of figure labels (e.g. `["Figure 1"]`).

Paths in `artifact_paths` may point to files under `datasets/releases/<release_id>/` or `datasets/runs/<eval_name>/`. Tables and figures must be produced by scripts from those artifacts (autogenerated from release).

## Repro block (top of draft)

The draft document (`papers/Px/DRAFT.md`) must start with a **Reproducibility** block (after the title and optional one-line abstract) that gives **exact commands** to regenerate the draft’s tables and figure. Example:

```markdown
# Paper title

**Reproducibility.** From repo root, with `PYTHONPATH=impl/src` and `LABTRUST_KERNEL_DIR=kernel`:
- Table 1: `python scripts/export_e3_table.py` (consumes `datasets/releases/p0_e3_release/` or last E3 run).
- Table 2: `python scripts/export_foo_table.py --release p0_e3_release`.
- Figure 1: `python scripts/plot_bar.py --input datasets/runs/.../summary.json --out docs/figures/fig1.png`.
```

This block ensures reviewers can reproduce results without hunting through runbooks.

## Two tables + one figure

- At least **two tables** in the draft must be autogenerated from a release (or from `datasets/runs/` produced by a documented script).
- At least **one figure** must be autogenerated the same way.
- Script names and invocation must appear in the repro block and (as needed) in `EVALS_RUNBOOK.md` or the paper README.

## Limitations

The draft must include a **Limitations** (or **Non-goals**) section that:

- Avoids overclaim (e.g. no certification claim for P7; no “full determinism” for P3).
- States determinism levels explicitly (e.g. replay levels + nondeterminism detection, not bit-identical hardware replay).
- States restricted auditability where applicable (e.g. redaction, evaluator/regulator modes).

See `STATE_OF_THE_ART_CRITERIA.md` (Writing / No overclaim) for portfolio-wide wording.
