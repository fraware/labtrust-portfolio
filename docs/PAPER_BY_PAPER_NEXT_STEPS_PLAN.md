# Paper-by-paper next steps: implementation plan

This plan turns the "Paper-by-paper next steps" into concrete implementation tasks. Order follows: core-kernel (P0, P1, P3, P4, P7) then conditional (P2, P5, P6, P8). Each section lists tasks, artifact targets, and kill criteria to enforce.

## Current state (post-implementation)

**Portfolio status:** All nine papers (P0–P8) are at **Draft** stage. Phase 3 (submission-readiness) checklist passed 2025-03-11 for each; see `papers/Px_*/PHASE3_PASSED.md`. **Next:** Submission prep per [PRE_SUBMISSION_CHECKLIST.md](PRE_SUBMISSION_CHECKLIST.md): tag release, final pass (repro block, claim-evidence, venue format), publishable seeds. Regenerate tables/figures with `python scripts/generate_paper_artifacts.py --paper Px` or `--paper all`; paste from `papers/Px_*/generated_tables.md` into DRAFT.md as needed.

The following are in place: **verification_mode** in EVIDENCE_BUNDLE and VERIFICATION_MODES; **p0_conformance_summary** script and portfolio release layout; **P1_TRACE_DERIVABILITY.md** and gatekeeper test; **witness_slice** and **witness_slices** in replay_eval summary; **L1** explicit wording in REPLAY_LEVELS and P3 draft; **K3**, **K1**, **K4**, **K0**, **K7** in limitations; **lab_profile_v0** as reference organism; **scoring_proof** in maestro_antigaming; **audit_bundle.py --release**; conditional headers and trigger refs (P2, P5, P6, P8); **claims.yaml** with evidence mapping for all papers; repro blocks at top of DRAFTs; Phase 3 checklist in STATE_OF_THE_ART_CRITERIA section 3. **Excellence metrics:** Eval scripts write optional `excellence_metrics` to summary JSONs; run `python scripts/export_excellence_summary.py` to print a one-line summary per paper. See [STANDARDS_OF_EXCELLENCE.md](STANDARDS_OF_EXCELLENCE.md).

**Conventions:**
- **Artifact path** = file or dir to create/update.
- **Kill criteria** = condition that, if violated, forces demotion or scope change (document in AUTHORING_PACKET / draft).
- **Trigger proof** (conditional papers) = evidence that must exist in release for the module to remain justified.

## Success criteria (portfolio-level)

- Every **core-kernel** paper (P0, P1, P3, P4, P7) has: claim table (claims.yaml), evidence mapping, at least two tables + one figure from release, limitations, repro block at top of draft, and kill criteria documented.
- Every **conditional** paper (P2, P5, P6, P8) has: explicit trigger proof in release (or documented "not met" and scope narrowed), plus same draft bar when promoted.
- No admissibility/conformance condition is normative unless it is computable from logged fields (P0 K0).
- Conditional modules remain justified by trigger proofs; if a trigger is not met, paper and docs state "conditional / optional" and negative result clearly.
