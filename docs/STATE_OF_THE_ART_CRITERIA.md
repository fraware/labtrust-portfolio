# State-of-the-Art Criteria (Portfolio-Wide)

All papers in the LabTrust portfolio adhere to the following criteria for rigor and reproducibility. For **optional excellence metrics** beyond this bar (effect size, statistical tests, per-paper metrics, artifact badge readiness), see [STANDARDS_OF_EXCELLENCE.md](STANDARDS_OF_EXCELLENCE.md).

## Peer-review bar

For a paper to be **peer-review ready** (submission grade), the following are non-negotiable:

- **Tables and figures:** Every table and figure cited in the draft must be **autogenerated by a documented script** from release or run artifacts. No "optional" or "no dedicated script" figures; each has a script name in the repro block and produces output from `datasets/` or kernel.
- **Sample size:** Publishable result tables use **at least 20 seeds** (or a justified exception documented in the run manifest and limitations). CI may use fewer for speed; the run manifest must record seeds and settings.
- **Run manifest:** Every eval summary JSON includes a `run_manifest` (seeds, scenario_ids, fault settings, script) so results are reproducible and auditable.
- **Sensitivity or power (optional excellence):** Report CI width or post-hoc power for key comparisons; or document a sensitivity run at N=30 (e.g. via `scripts/sensitivity_seed_sweep.py`).
- **Raised bar (optional):** Non-vacuous conditional triggers (P8: `--non-vacuous` with `stress_selection_policy` in `run_manifest`; publishable dual-scenario `regime_stress_v0` + `regime_stress_v1` via `run_paper_experiments.py --paper P8`; `verify_p8_meta_artifacts.py`); regression reported or explicitly N/A with reason (P5: regression_skipped_reason, fallback OLS); L1 twin replay or documented design (P3: --l1-twin, L2 design subsection); multi-scenario or multi-step evidence (P2: three scenarios default including rep_cps_scheduling_v0, delay_fault_prob sweep, optional drop_completion_prob sweep via `--drop-sweep`, per_scenario summaries, scheduling_dependent_eval, excellence_metrics with comparison statistics, freshness_replay_evidence, sybil_vs_spoofing_evidence, messaging_sim, dynamic_aggregation_series, --aggregation-steps); validator beyond allow-list or real-LLM artifact (P6: safe_args, export_llm_redteam_table, export_p6_reproducibility_table, export_p6_layer_attribution, export_p6_failure_analysis); multi-profile or explicit review scope (P7: two assurance-pack instantiations, PONR without heuristics).
- **Formal verification:** At least two Lean wedges with machine-checked proofs (W1, W2); W3 evidence-bundle verifier has formal spec and soundness statements with proofs pending (see [formal/lean/README.md](formal/lean/README.md)). Python test harness aligns impl to spec.
- **Evidence mapping:** Every claim in `claims.yaml` references concrete artifact_paths and at least one table_id or figure_id produced by a script.
- **Limitations:** No overclaim; determinism level (replay, not full hardware determinism), auditability scope, and conditional/trigger status stated explicitly.

## Methodology

- **Hypothesis/claims:** Each paper states clear claims (C1–C4 or equivalent) that are falsifiable.
- **Kill criteria:** Where applicable, explicit kill criteria define failure (e.g. cannot define measurable bounds; does not beat baseline; safety regression).
- **Metrics:** Primary and secondary metrics are defined and reported (e.g. pass/fail, latency p95, variance, detection rate).
- **Baselines:** Comparisons against at least one baseline (e.g. naive, unsecured, fixed regime) where the method should outperform or match.

## Reproducibility

- **Artifacts:** All experiments reproducible from `datasets/` with release manifest; kernel schemas and docs are versioned. Evidence bundle schema requires `verification_mode` (public | evaluator | regulator). Each eval writes a summary JSON under `datasets/runs/<eval_name>/`: P0 e3_summary.json, p0_e3_variance.json, e2_redaction_demo, conformance.json per run; portfolio release `datasets/releases/portfolio_v0.1/p0_conformance_summary.json` (from build_p0_conformance_summary.py); P1 contracts_eval/eval.json (ablation, ablation_by_class, detection_metrics_by_class, excellence_metrics with per_class_ci95 every run; --baseline adds violations_would_apply_without_validator), scale_test.json (--scale-test), optional scale_sweep.json (--scale-sweep), stress_results.json (async delay/skew/reorder sweep from contracts_async_stress.py), transport_parity.json (parity_ok_all, per_sequence[], parity_confidence; contracts_transport_parity.py); P2 rep_cps_eval/summary.json, safety_gate_denial.json; P3 replay_eval/summary.json (root_cause_category, witness_slice, overhead_curve, corpus_space_summary, per_trace localization/size fields); P4 maestro_fault_sweep/multi_sweep.json, maestro_antigaming/antigaming_results.json (scoring_proof), bench/maestro/adapter_costs.json; P5 scaling_eval/heldout_results.json (per_scenario_baseline_mae, trigger_met); P6 llm_eval/red_team_results.json, confusable_deputy_results.json, e2e_denial_trace.json, adapter_latency.json, baseline_comparison.json, baseline_benign.json, denial_trace_stats.json; P7 assurance_eval/results.json, audit_bundle.py (--release); P8 `meta_eval/comparison.json` (`schema_version`, `collapse_paired_analysis`, `meta_non_worse_collapse`, `meta_strictly_reduces_collapse`, `trigger_met`, optional `stress_selection_policy`), `collapse_sweep.json` (`schema_version`), optional `meta_eval/scenario_regime_stress_v1/comparison.json`.

## Artifacts

- **Kernel:** Schemas and normative docs under `kernel/` with version suffix (e.g. v0.1).
- **Code:** Implementation in `impl/` or `bench/`; trace corpora and expected outcomes machine-readable (JSON).
- **Evidence:** Runs produce trace, MAESTRO report, evidence bundle, release manifest; conformance checker validates.

## 2. Draft conversion (Eval → Draft)

A paper can move to **Draft** only when it satisfies the following. This is the standards-of-excellence spine.

### 2.1 Portfolio-wide Draft conversion checklist

1. **Claim table** — `papers/Px/claims.yaml` with **3–4 claims max**. Each claim has an id (e.g. C1–C4), a short statement, and evidence mapping (see below).
2. **Evidence mapping** — Each claim references:
   - **release_id** (e.g. `p0_e3_release`) tying results to a specific dataset release.
   - **artifact_paths** (e.g. `datasets/releases/p0_e3_release/evidence_bundle.json`, or paths under `datasets/runs/`).
   - **table_id** / **figure_id** (e.g. Table 1, Figure 1) that present the evidence in the draft.
3. **Two tables + one figure (minimum)** — At least two tables and one figure **autogenerated from release** by scripts under `scripts/`. For peer-review submission, each paper should also include a **system or methodology overview figure** (Figure 0: pipeline, flow, or architecture diagram). Full visual set per paper: `docs/VISUALS_PER_PAPER.md`. Every table and figure must have a script; no placeholder or "optional" figures.
4. **Limitations** — A limitations section that matches the discipline: no overclaim, explicit determinism levels (replay levels + nondeterminism detection, not full hardware determinism), restricted auditability where applicable.
5. **Repro block at top of draft** — The draft starts with a short **Reproducibility** block giving **exact commands** to reproduce its tables (and figure) from the repo root, including env (e.g. `PYTHONPATH=impl/src`, `LABTRUST_KERNEL_DIR=kernel`).

The claim table schema is documented in `docs/DRAFT_CONVERSION_CHECKLIST.md`; see `papers/P0_MADS-CPS/claims.yaml` for an example.

## Writing

- **Outline:** Draft follows AUTHORING_PACKET outline per paper.
- **Backing:** Every claim has an explicit "backed by" sentence (artifact or experiment).
- **No overclaim:** No certification claim (P7); no full hardware determinism (P3); containment not elimination (P6).

## 3. Submission readiness (Phase 3 / hostile reviewer checklist)

Before submission, each paper must pass the following. Mandatory before marking a paper Draft-complete or submitting.

1. **Claim-evidence matrix complete** — Every claim in `claims.yaml` has artifact_paths and at least one table_id or figure_id; no claim is unsupported by a script-generated table or figure or kernel artifact.
2. **Repro steps under 20 minutes** — From a fresh clone, the minimal run (one scenario, minimal seeds) to regenerate the draft's tables and Figure 1 (and optionally Figure 0) is documented at the top of DRAFT.md and is executable in under ~20 minutes with the stated env (PYTHONPATH, LABTRUST_KERNEL_DIR).
3. **Variance rules followed** — Where the draft reports means or CIs, the run manifest (seeds, scenario, fault settings) is stated; for publishable tables use at least 20 seeds unless justified in limitations. See `docs/REPORTING_STANDARD.md`.
4. **No redefinition of another paper's kernel** — The draft does not redefine conformance tiers, admissibility, PONR, trace format, or evaluation schema; it cites the owning paper and kernel. See `docs/INTEGRATION_CONTRACTS.md`.
5. **Overclaim check** — No certification claim (P7); no "full determinism on hardware" (P3); replay levels and nondeterminism detection stated; restricted auditability where applicable; conditional papers (P2, P5, P6, P8) state trigger and scope in limitations.
6. **Repro block lists every script** — The Reproducibility block at the top of DRAFT.md lists the exact command for Figure 0, Table 1, Table 2, and Figure 1 (or the paper's actual set); no table or figure is "optional" or without a script.
