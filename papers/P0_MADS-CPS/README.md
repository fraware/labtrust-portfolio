# MADS-CPS: A Normative Minimum Assurance Bar for Agentic Cyber-Physical Workflows

P0 defines the normative minimum assurance bar for agentic CPS: conformance tiers (E1), redaction (E2), and replay-link variance (E3). Draft and claim table: [DRAFT.md](DRAFT.md), [claims.yaml](claims.yaml). Tables and figures: [VISUALS_PER_PAPER.md](../docs/VISUALS_PER_PAPER.md), [RESULTS_PER_PAPER.md](../docs/RESULTS_PER_PAPER.md). Outline: [AUTHORING_PACKET.md](AUTHORING_PACKET.md).

**E1 (Thin-slice conformance):** The conformance checker (`labtrust_portfolio check-conformance`) demonstrates Tier checks: missing artifact yields Tier 1 FAIL; present and validated yields Tier 2 PASS when replay and verification flags are ok; Tier 3 adds PONR coverage (trace contains required PONR-aligned task_end events). See `docs/VALIDATING_A_RUN.md` and `impl/src/labtrust_portfolio/conformance.py`.

**E2 (Redaction):** Run `python scripts/e2_redaction_demo.py --out datasets/runs/e2_redaction_demo`. Produces `trace_redacted.json` and `evidence_bundle_redacted.json` (verification_mode, redaction_manifest). The redacted trace is for audit only (payloads replaced by refs); it is not replayed, and the bundle has replay_ok false. See `kernel/mads/VERIFICATION_MODES.v0.1.md`.

**E3 (Replay link):** Run `python scripts/produce_p0_e3_release.py --runs 20` (with LABTRUST_KERNEL_DIR and PYTHONPATH set). This produces runs under `datasets/runs/e3/<scenario>/seed_<n>/`, `e3_summary.json`, `p0_e3_variance.json`, and optionally the release to `datasets/releases/p0_e3_release`. Release paths exist only when run without `--no-release`; tables can be regenerated from `e3_summary.json` in `datasets/runs/` (export_e3_table.py). Tables and figures: `export_e3_table.py` (full table), `export_e3_table.py --compact` (per-scenario variance + CI), `export_e2_admissibility_matrix.py` (E2 redaction admissibility matrix), `plot_e3_latency.py` (E3 p95 latency by scenario). Conformance summary: `python scripts/build_p0_conformance_summary.py` writes `datasets/releases/portfolio_v0.1/p0_conformance_summary.json`. Evidence bundle schema requires `verification_mode` (public | evaluator | regulator). See `docs/P0_CONFORMANCE_SUMMARY_SPEC.md`. To run P0 together with other papers: `python scripts/run_paper_experiments.py --paper P0`.
