# P2 — REP-CPS

P2 defines a safety-gated CPS profile for authenticated sensitivity-sharing (typed variables, freshness, rate limits, provenance, robust aggregation) integrated with MADS safety gates. Draft and claims: [DRAFT.md](DRAFT.md), [claims.yaml](claims.yaml). Reviewer hardening: [REVIEWER_ATTACK_SURFACE_LEDGER.md](REVIEWER_ATTACK_SURFACE_LEDGER.md), [REVIEWER_REBUTTAL_MATRIX.md](REVIEWER_REBUTTAL_MATRIX.md). Tables and figures: [VISUALS_PER_PAPER.md](../../docs/VISUALS_PER_PAPER.md), [RESULTS_PER_PAPER.md](../../docs/RESULTS_PER_PAPER.md), [generated_tables.md](generated_tables.md). Outline: [AUTHORING_PACKET.md](AUTHORING_PACKET.md).

**Eval:** Publishable default uses **20 seeds**, scenarios **toy_lab_v0, lab_profile_v0, rep_cps_scheduling_v0**, and delay sweep; output in `datasets/runs/rep_cps_eval/summary.json`. The **scheduling-dependent** scenario demonstrates task-level effect of the gate under spoof stress (`scheduling_dependent_eval`). Tables 1–7 and threat-evidence blocks: `python scripts/export_rep_cps_tables.py`. Figures: `plot_rep_cps_summary.py` (per-scenario + global), `plot_rep_cps_gate_threshold.py`, `plot_rep_cps_dynamics.py`, `plot_rep_cps_latency.py`. Figure 0 source and camera-ready renders: `export_p2_rep_profile_diagram.py` with optional `--render-png|--render-svg|--render-pdf`. Optional `--aggregation-steps 5` for Table 4.

**Conditional scope:** On toy/lab scenarios, adapter parity with centralized is **scoped** (scheduling does not read aggregate). On `rep_cps_scheduling_v0`, gated aggregate can change **tasks_completed**. See [CONDITIONAL_TRIGGERS.md](../../docs/CONDITIONAL_TRIGGERS.md), [TRIGGER_P2_REP_CPS.md](../../docs/TRIGGER_P2_REP_CPS.md). Run: `python scripts/run_paper_experiments.py --paper P2`.
