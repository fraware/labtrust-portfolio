# P2 — REP-CPS

P2 defines a safety-gated CPS profile for authenticated sensitivity-sharing (typed variables, freshness, rate limits, provenance, robust aggregation) integrated with MADS safety gates. Draft and claims: [DRAFT.md](DRAFT.md), [claims.yaml](claims.yaml). Tables and figures: [VISUALS_PER_PAPER.md](../docs/VISUALS_PER_PAPER.md), [RESULTS_PER_PAPER.md](../docs/RESULTS_PER_PAPER.md), [generated_tables.md](generated_tables.md). Outline: [AUTHORING_PACKET.md](AUTHORING_PACKET.md). **Eval:** Publishable run uses 20 seeds, two scenarios (toy_lab_v0, lab_profile_v0), and delay sweep; output in `datasets/runs/rep_cps_eval/summary.json`. Tables 1–7 and Figures 1–2: run `python scripts/export_rep_cps_tables.py` (Tables 1–7 from summary.json), `python scripts/plot_rep_cps_summary.py` (Figure 1), `python scripts/plot_rep_cps_latency.py` (Figure 2). Profile ablation in summary.json `profile_ablation` (Table 6); latency/cost (Table 5); resilience envelope (Table 7); safety-gate campaign (pass/deny counts, denial_trace_recorded). Optional `--aggregation-steps 5` for multi-step convergence (Table 4). Conditional paper: in the evaluated scenario, sensitivity sharing does not materially change tasks_completed; the contribution is the profile and MAESTRO-compatible harness. Limitations describe a future scenario where sensitivity would matter. See [CONDITIONAL_TRIGGERS.md](../docs/CONDITIONAL_TRIGGERS.md) and [TRIGGER_P2_REP_CPS.md](../docs/TRIGGER_P2_REP_CPS.md). To run: `python scripts/run_paper_experiments.py --paper P2`. Key results (adapter parity, reduced observed bias, safety gate, latency, resilience envelope) are in Section 5 of DRAFT and in [RUN_RESULTS_SUMMARY.md](../../datasets/runs/RUN_RESULTS_SUMMARY.md).
