# P8 — Meta-Coordination

P8 contributes **auditability** (every `regime_switch` in the trace with a reason), **PONR invariance**, and a publishable paired evaluation under compound stress. Evidence is emitted as `comparison.json` with **`schema_version`: `p8_meta_eval_v0.3`**: non-inferiority on paired collapse counts (`meta_non_worse_collapse`), strict count reduction when it holds (`meta_strictly_reduces_collapse`), McNemar and Wilson CIs in `collapse_paired_analysis`, and explicit **outcome attribution** when `--fallback-adapter retry_heavy` is used (meta outcomes after a switch are scored on the fallback run).

**Stress policy.** Non-vacuous runs use `--non-vacuous` with `--non-vacuous-select max_drop_any_collapse` (largest drop in the sweep grid where the fixed arm still collapses at least once). **`--calibration-seeds`** restrict the sweep grid to calibration seeds; **`--seeds`** are the held-out evaluation seeds (portfolio default: calibration `1–10`, evaluation `11–30`).

**Scenarios and artifacts.** Primary: `datasets/runs/meta_eval/comparison.json` and `collapse_sweep.json`. Secondary: `datasets/runs/meta_eval/scenario_regime_stress_v1/`. Optional external-validity lab slice: `datasets/runs/meta_eval/scenario_lab_profile_v0/comparison.json` when that scenario is run. Robustness matrix: `datasets/runs/meta_eval/robustness_campaign/campaign_summary.json` from `scripts/p8_robustness_campaign.py --strict-publishable`.

Draft and claims: [DRAFT.md](DRAFT.md), [claims.yaml](claims.yaml). Verifier: `python scripts/verify_p8_meta_artifacts.py` (use `--strict-publishable` for submission checks). One-shot portfolio eval: `python scripts/run_paper_experiments.py --paper P8`.
