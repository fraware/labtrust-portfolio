## Regenerate tables and LaTeX (after changing held-out JSON)

From repo root (bash):

```bash
export PYTHONPATH=impl/src
export LABTRUST_KERNEL_DIR=kernel
python scripts/export_scaling_tables.py \
  --out papers/P5_ScalingLaws/generated_tables.md \
  --out-tex-dir papers/P5_ScalingLaws/neurips2026/tables_tex
```

## Reviewer interpretation (one paragraph)

Empirical **coordination-scaling patterns** on a **7,200-row** grid; **scenario-heldout** `trigger_met` is **false** against the admissible **num-tasks bucket** baseline; **family-heldout** `trigger_met` is **true**; **coordination-tax proxy** is much more predictable than throughput or latency-derived **error amplification** under scenario-heldout (Main Table 4; per-target MAE scales differ). Recommendation metrics and log-log **`scaling_fit`** are **exploratory** only — not deployment-ready.

See `../CLAIM_LOCK_NEURIPS2026.md` Tier D for language **not** to use when writing or reviewing.
